# Βάση Δεδομένων Νοσοκομείου "Υγειόπολις"
## Εξαμηνιαία Εργασία, Βάσεις Δεδομένων, 6ο Εξάμηνο 2025-2026

## Παραδοχές για τα ER και Relational Diagrams
- Λόγω GDPR (EU) 2016/679, εικόνες επιτρέπεται να παρέχονται μόνο από τις εξής οντότητες του νοσοκομείου:
  -	Προσωπικό (ιατροί, νοσηλευτές, διοικητικό προσωπικό)
  -	Τμήματα Νοσοκομείου
  -	Χώροι ιατρικών πράξεων/επεμβάσεων
 
## Γενικές Παραδοχές
- Όσον αφορά τον υπολογισμό του κόστους νοσηλείας, η επιπλέον χρέωση ανά ημέρα είναι 50 €.
- Στο τελικό κόστος νοσηλείας περιλαμβάνεται μόνο το βασικό κόστος της νοσηλείας και πιθανές επιπλέον ημερήσιες χρεώσεις.
- Ως προς τις αξιολογήσεις ιατρικής φροντίδας, ένας ασθενής αξιολογεί τους γιατρούς που του συνταγογράφησαν συνολικά κι όχι προσωπικά.
- Οι ηλικίες των ανθρώπων στη βάση μπορεί να εμφανίζουν διαφορά με την πραγματικότητα, λόγω τυχαίου data generation.

## Παραδοχές για τα Queries
- Στο **Q5** παρουσιάζουμε τους ιατρούς που έχουν τον μέγιστο αριθμό επεμβάσεων. 
- Στο **Q7** ταξινομούμε βάσει αριθμού αλλεργικών ασθενών σε φθίνουσα σειρά.
- Στο **Q12** ως έξοδο υπολογίζουμε το προσωπικό που έχει αναλάβει τις υπό εξέταση βάρδιες και το οποίο πληροί όλες τις προδιαγραφές που δίνονται από την εκφώνηση. Γίνεται ανάλυση ανά κλάση (ιατρός, νοσηλευτής, διοικητικό προσωπικό) όσο και ανά υποκλάση προσωπικού (ιατροί ανά ειδικότητα, νοσηλευτές ανά βαθμίδα, διοικητικό προσωπικό ανά ρόλο). Η τελευταία στήλη δίνει το πλήθος των υπαλλήλων που υπάρχουν ανά υποκλάση προσωπικού.
- Στο **Q14** υποθέτουμε ότι η κατηγορία ICD-10 διαγνώσεων είναι ο πρώτος χαρακτήρας των αλφαριθμητικών της στήλης ICD_code του πίνακα ICD κωδικών (π.χ. Ο κωδικός Α04.2 ανήκει στην κατηγορία Α.)

## Δημιουργία Dummy Data
Η δημιουργία των dummy data που εισήχθησαν στη βάση έγινε με χρήση της γεννήτριας δεδομένων mockaroo και, όπου απαιτούνταν να ικανοποιούνται περιορισμοί (constraints, triggers) τα οποία δεν ήταν δυνατόν να ικανοποιηθούν με τη χρήση του mockaroo, έγινε χρήση μοντέλων Τεχνητής Νοημοσύνης, κυρίως του μοντέλου Gemini.

## Φόρτωση των δεδομένων στη βάση - ΣΗΜΑΝΤΙΚΟ
Καθώς στη βάση περιλαμβάνονται εγγραφές με ελληνικούς χαρακτήρες, για να μπορούν να τυπωθούν στο terminal χωρίς να εμφανίζονται ερωτηματικά θα πρέπει, πριν την εισαγωγή στη βάση να εκτελεστεί η εντολή:
```
chcp 65001
```

Για τη δημιουργία της βάσης:
```
CREATE DATABASE ygeiopolis
CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci; 
```

Για την χρήση της βάσης:
```
USE ygeiopolis;
```

Επιπλέον, όλα τα αρχεία .csv πρέπει να τοποθετηθούν σε φάκελο με το εξής μονοπάτι: **C:/xampp/htdocs/csv_imports/**

Για τη λειτουργία της εφαρμογής, η βάση δεδομένων πρέπει να είναι ονοματισμένη **ygeiopolis**, οπως υποδεικνύεται στο db_connection.php
